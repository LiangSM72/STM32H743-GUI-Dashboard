# STM32H743 GUI Dashboard - 開發環境配置指南

本文檔將詳細介紹如何在 Windows / Linux / macOS 上配置 STM32H743 開發環境。

## 📋 系統需求

- **Windows 10/11** / **Ubuntu 20.04+** / **macOS 10.15+**
- **VS Code** (Latest)
- **Git** 已安裝
- **USB 轉 UART** 串口線 (用於調試)
- **ST-Link v2 調試器** (用於燒錄)

---

## 🛠️ 第一步：安裝必要工具

### 1. 安裝 VS Code

下載地址: https://code.visualstudio.com/

安裝完成後，安裝以下擴展：
- **C/C++** (Microsoft)
- **Cortex-Debug** (Arm)
- **Git Graph** (可選)
- **Serial Monitor** (可選)

### 2. 安裝 ARM GCC 工具鏈

#### Windows:
```bash
# 下載 (版本: arm-gnu-toolchain-12.2.rel1-mingw-w64-i686-arm-none-eabi)
# 地址: https://developer.arm.com/tools-and-software/open-source-software/developer-tools/gnu-toolchain/gnu-rm/downloads
# 選擇 .exe 安裝程序
# 安裝到: C:\Program Files (x86)\GNU Arm Embedded Toolchain
```

#### Linux (Ubuntu):
```bash
sudo apt update
sudo apt install arm-none-eabi-gcc arm-none-eabi-gdb arm-none-eabi-newlib
```

#### macOS:
```bash
brew install arm-none-eabi-gcc
```

驗證安裝：
```bash
arm-none-eabi-gcc --version
```

### 3. 安裝 STM32CubeMX (可選)

下載地址: https://www.st.com/en/development-tools/stm32cubemx.html

用於生成 HAL 庫代碼（本項目已包含）

### 4. 安裝 OpenOCD 調試器

#### Windows:
```bash
# 下載: https://github.com/openocd-org/openocd/releases
# 解壓到: C:\openocd
```

#### Linux:
```bash
sudo apt install openocd
```

#### macOS:
```bash
brew install openocd
```

### 5. 安裝 ST-Link 驅動

#### Windows:
下載: https://www.st.com/en/development-tools/stsw-link009.html

#### Linux:
```bash
sudo apt install libusb-1.0-0-dev
```

#### macOS:
```bash
brew install libusb
```

## 📂 第二步：項目配置

### 1. 克隆項目

```bash
git clone https://github.com/LiangSM72/STM32H743-GUI-Dashboard.git
cd STM32H743-GUI-Dashboard
```

### 2. 編輯 Makefile

打開 `Makefile`，修改工具鏈路徑：

#### Windows:
```makefile
ARM_TOOLCHAIN_PATH = C:\\Program Files (x86)\\GNU Arm Embedded Toolchain\\bin
```

#### Linux/macOS:
```makefile
ARM_TOOLCHAIN_PATH = /usr/bin
```

### 3. 編譯項目

```bash
make clean
make all
```

期望輸出:

## 🔧 第三步：VS Code 配置

### 1. 配置 IntelliSense

VS Code 會自動讀取 `.vscode/c_cpp_properties.json`

如果有紅色波浪線，按 `Ctrl+Shift+P` → "C/C++: Edit Configurations"

### 2. 配置編譯任務

按 `Ctrl+Shift+B` 應該看到:
- Build (編譯)
- Clean (清理)
- Flash (燒錄)

### 3. 配置調試

確保已安裝 **Cortex-Debug** 擴展

---

## 💾 第四步：燒錄固件

### 前置準備

1. **連接 ST-Link 調試器到開發板**
   - SWCLK → PA14
   - SWDIO → PA13
   - GND → GND

2. **連接 USB 轉 UART 到開發板**
   - TX → PA9
   - RX → PA10
   - GND → GND

3. **給開發板供電** (3.3V USB)

### 燒錄方法

#### 方法 1: 命令行
```bash
make flash
```

#### 方法 2: VS Code
按 `Ctrl+Shift+B` → 選擇 "flash"

#### 方法 3: 手動使用 st-flash
```bash
st-flash write build/STM32H743_GUI.bin 0x08000000
```

期望輸出:
```
st-flash 1.7.0
Flash page at 0x08000000 erased.
Flashing file: build/STM32H743_GUI.bin [100%]
Flashing done.
```

## 🐛 第五步：調試

### 1. 打開串口監視器

使用任意串口工具 (如 PuTTY、Minicom)：
- **波特率**: 115200
- **數據位**: 8
- **停止位**: 1
- **校驗**: 無

### 2. 在 VS Code 中調試

1. 連接 ST-Link 調試器
2. 按 `F5` 啟動調試
3. 在代碼中設置斷點 (按 `F9`)
4. 使用調試命令:
   - `F5` - 繼續
   - `F6` - 單步
   - `F10` - 跳過
   - `Shift+F5` - 停止調試

### 3. 查看調試輸出

在 DEBUG CONSOLE 中查看 printf 輸出

## 🆘 常見問題

### Q1: 編譯錯誤 "cannot find -lstdc++"

**解決**: 安裝 ARM GCC 工具鏈

```bash
# Windows: 下載並安裝 .exe
# Linux: sudo apt install arm-none-eabi-g++
# macOS: brew install arm-none-eabi-gcc
```

### Q2: 燒錄失敗 "No ST-Link devices found"

**解決**:
1. 檢查 USB 連接
2. 安裝 ST-Link 驅動
3. 運行 `st-info --probe`

### Q3: 串口沒有輸出

**解決**:
1. 檢查波特率 (應為 115200)
2. 檢查 TX/RX 連接
3. 重新上電開發板
